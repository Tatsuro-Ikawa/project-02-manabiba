# GitHubバックアップガイド

現在のコードをGitHubにバックアップする手順です。

## 📋 バックアップ手順

### ステップ1: 変更をステージング（追加）

すべての変更をステージングエリアに追加します：

```bash
# すべての変更を追加
git add .

# または、特定のファイルのみ追加する場合
git add docs/
git add firestore.rules
git add src/
```

### ステップ2: コミット（変更を記録）

変更をコミット（記録）します：

```bash
# コミットメッセージを付けてコミット
git commit -m "バックアップ: 外部仕様書テンプレートとビジュアルエディターガイドを追加"
```

**コミットメッセージの例**:
- `"バックアップ: 設計書テンプレート追加"`
- `"バックアップ: 現状のコードを保存"`
- `"feat: 外部仕様書テンプレート追加"`

### ステップ3: GitHubにプッシュ（アップロード）

リモートリポジトリ（GitHub）にプッシュします：

```bash
# 現在のブランチ（master）をプッシュ
git push origin master

# または、mainブランチの場合
git push origin main
```

### ステップ4: 確認

GitHubのリポジトリページで、変更が反映されているか確認します。

## 🔍 現在の状態

現在のGitの状態：
- **ブランチ**: master
- **未コミットの変更**: 多数のファイル
- **未追跡ファイル**: 新規追加されたファイル（docs/, firestore.rules など）

## ⚠️ 注意事項

### .gitignoreの確認

機密情報や不要なファイルがコミットされないように、`.gitignore`を確認してください：

```bash
# .gitignoreファイルを確認
cat .gitignore
```

**通常、以下のファイルは除外すべき**:
- `node_modules/`
- `.env`（環境変数ファイル）
- `.next/`（Next.jsのビルドファイル）
- `*.log`（ログファイル）

### 機密情報の確認

以下のような機密情報が含まれていないか確認：
- APIキー
- パスワード
- 個人情報
- Firebase設定ファイル（機密情報が含まれる場合）

## 📝 推奨されるコミットメッセージ

### バックアップ用
```
バックアップ: [日付] 現状のコードを保存
```

### 機能追加用
```
feat: 外部仕様書テンプレート追加
feat: ビジュアルエディター学習ガイド追加
```

### ドキュメント追加用
```
docs: 設計書テンプレート追加
docs: GitHubバックアップガイド追加
```

## 🚀 一括実行コマンド

すべての変更を一度にコミット・プッシュする場合：

```bash
# 1. すべての変更を追加
git add .

# 2. コミット
git commit -m "バックアップ: 現状のコードを保存"

# 3. GitHubにプッシュ
git push origin master
```

## 🔄 ブランチの確認

現在のブランチを確認：

```bash
git branch
```

リモートブランチを確認：

```bash
git branch -r
```

## 📚 よく使うGitコマンド

### 状態確認
```bash
git status          # 現在の状態を確認
git log             # コミット履歴を確認
git diff            # 変更内容を確認
```

### 変更の取り消し
```bash
# ステージングを取り消し
git restore --staged <ファイル名>

# 変更を取り消し（注意：変更が失われます）
git restore <ファイル名>
```

### リモートの確認
```bash
# リモートリポジトリの確認
git remote -v
```
