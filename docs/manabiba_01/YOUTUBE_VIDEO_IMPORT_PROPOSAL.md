# YouTube 動画 URL から付帯情報を取得する実装案

**※ 実装済み**: 本提案に基づき、`/api/youtube-oembed`・`src/lib/youtube.ts`（Shorts 対応）・`LatestVideosEditModal` の「URLから情報を取得」を実装済み。記事 URL からは OGP 取得（下記「関連」）で同様の自動入力を実装済み。

## 0. 既存コードでの実装状況

プロジェクト内には、**YouTube 用の処理が一部すでにあります**。場所は `**src/components/VideoLinkPlayer.tsx`** です。

### 0.1 既にある処理


| 内容                 | 実装箇所                                                | 説明                                                                                                                                                               |
| ------------------ | --------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **動画 ID の抽出**      | `getVideoEmbedUrl` 内（59–64 行付近）                     | `url.includes('embed/')` のときは `url.split('embed/')[1]` で ID を取得。それ以外は `url.split('v=')[1]?.split('&')[0]` で `watch?v=XXX` 形式から ID を取得。`**youtu.be/XXX` 形式は未対応**。 |
| **埋め込み URL の組み立て** | 同上                                                  | 取得した videoId から `https://www.youtube.com/embed/${videoId}` を返している。                                                                                               |
| **サムネイル URL の利用**  | `sampleVideos`（19–40 行付近）                           | サンプルデータで `https://img.youtube.com/vi/{videoId}/maxresdefault.jpg` をハードコードで使用。**URL や videoId から組み立てる関数はない**。                                                     |
| **動画タイプ**          | `VideoLink` の `type: 'youtube' | 'vimeo' | 'other'` | YouTube / Vimeo / その他 を区別。                                                                                                                                       |


### 0.2 まだない処理

- `**youtu.be/VIDEO_ID` 形式**からの動画 ID 抽出。
- **URL または videoId からサムネイル URL を組み立てる共通関数**（`img.youtube.com/vi/{id}/hqdefault.jpg` など）。
- **URL 貼り付けでタイトル・サムネイルを自動取得する処理**（oEmbed や API の呼び出し）。  
  - カスタム URL 入力（`handleCustomUrlSubmit`）では、URL をそのまま渡しているだけで、タイトルは「カスタム動画」固定。YouTube のメタデータ取得は行っていない。
- **YouTube oEmbed を叩く API Route**（`app/api/` 配下に API ルートは現状なし）。

→ **「URL を貼り付けて付帯情報（タイトル・サムネイル）を取る」処理は、現状どこにもなく、VideoLinkPlayer は「再生用の embed URL の組み立て」と「サンプル用のサムネイル URL の利用」まで。**

## 1. リンクから入手できる情報

### 1.1 URL をパースして得られるもの（クライアントのみ・API 不要）


| 情報                  | 取得方法                                                                                                        |
| ------------------- | ----------------------------------------------------------------------------------------------------------- |
| **動画 ID (videoId)** | URL から正規表現などで抽出。例: `https://www.youtube.com/watch?v=ABC123` → `ABC123`、`https://youtu.be/ABC123` → `ABC123` |
| **サムネイル画像 URL**     | 動画 ID から以下の形式で組み立て可能（YouTube が提供する静的 URL）。API キー不要。                                                         |


**サムネイル URL のパターン（いずれも ID のみで構成）**

- `https://img.youtube.com/vi/{videoId}/maxresdefault.jpg` — 最大解像度（1280x720）。存在しない動画だと 404 になることがある。
- `https://img.youtube.com/vi/{videoId}/hqdefault.jpg` — 高画質（480x360）。多くの動画で利用可能。
- `https://img.youtube.com/vi/{videoId}/mqdefault.jpg` — 中画質（320x180）
- `https://img.youtube.com/vi/{videoId}/sddefault.jpg` — 640x480

→ **タイトル**は、クライアントだけで URL からは取れない（サーバーか oEmbed/API が必要）。

---

### 1.2 oEmbed で取得できる付帯情報（API キー不要）

YouTube は **oEmbed** を提供している。

- **エンドポイント例**: `https://www.youtube.com/oembed?url={YouTubeのURL}`
- **取得方法**: 上記 URL に GET でアクセスすると JSON が返る。
- **API キー**: 不要。

**返却される主な項目（oEmbed の固定スキーマ）**

| 項目 | 説明 | ない場合の扱い |
|------|------|----------------|
| `title` | 動画タイトル | 空欄 or 「なし」表示でよい |
| `thumbnail_url` | サムネイル画像の URL | 空欄。または動画 ID から `img.youtube.com/vi/{id}/hqdefault.jpg` で組み立ても可 |
| `thumbnail_width` / `thumbnail_height` | サムネイルの幅・高さ | 数値が返らない場合は空欄 or 表示しない |
| `author_name` | チャンネル名（制作者名） | 空欄 or 「なし」表示でよい |
| `author_url` | チャンネルページの URL | 空欄 or 表示しない |
| `html` | 埋め込み用 iframe の HTML | 取得のみでフォームには出さない運用でよい |

→ **項目名（属性名）は YouTube が決めた固定で、制作者が「独自の属性名」を追加することはできません。** 動画ごとに変わるのは **値**（タイトルやチャンネル名の中身）だけです。  
→ **値が返ってこない場合**（まれにフィールドが欠けている等）は、**空欄**や**「なし」**表示にして問題ありません。API 側で `?? ''` のようにフォールバックしておけば、編集画面では空の入力欄になります。

**注意**: ブラウザから直接 `youtube.com` の oEmbed に fetch すると、CORS でブロックされる可能性がある。そのため **Next.js の API Route（サーバー側）で oEmbed を叩き、結果だけをフロントに返す**形にすると安全。

**「制作者が属性名を個別に設定できるか」**

- **いいえ。** oEmbed の返却項目（`title`, `author_name`, `thumbnail_url` など）は **YouTube 側で決まった固定のスキーマ**です。動画のアップロード者（制作者）が「自分だけの属性名」を追加したり、項目名を変更したりすることはできません。
- 制作者が変えられるのは、**動画タイトル・説明文・サムネイル画像**などの **中身（値）** だけです。項目名はすべて YouTube の仕様どおりです。

---

### 1.3 YouTube Data API v3 を使う場合（オプション）

- **必要なもの**: Google Cloud でプロジェクト作成し、YouTube Data API v3 を有効化、API キー取得。
- **得られる情報**: タイトル・説明・サムネイル・公開日・チャンネル情報・長さなど、より詳細なメタデータ。
- **制約**: クォータ制限あり。無料枠でも 1 日あたり 1 万ユニット前後など、利用量の管理が必要。

→ **用途のイメージ**: YouTube の動画データそのものから、説明文・長さ・公開日など「一覧・検索・分析」に使う情報を取りたいときに Data API を検討する。現段階では「まず oEmbed で足りるか」を試し、必要になったら Data API を検討するのがおすすめ。

## 3. 推奨: 案 B（API Route + oEmbed）

「URL を貼り付けたら、タイトルとサムネイルをなるべく自動で入れたい」という要件を満たしつつ、API キー不要で済むため、**案 B** を推奨する。

### 3.1 想定フロー

1. 管理者が「最新動画の編集」モーダルで、YouTube の URL を「URL」欄に貼り付ける。
2. 「URL から情報を取得」ボタンを押す（または URL 入力の blur / 変更時に自動取得する、でも可）。
3. フロントは Next.js の API Route（例: `POST /api/youtube-oembed` や `GET /api/youtube-oembed?url=...`）を呼ぶ。
4. API Route 内で `https://www.youtube.com/oembed?url={エンコードしたURL}` に GET し、返ってきた JSON を受け取る。
5. API Route が `title`, `thumbnail_url` などを整形してフロントに返す。
6. フロントは受け取った値を「タイトル」「サムネイルURL」欄にセットする（既存の「URL」はそのまま）。

### 3.2 技術的なポイント

- **URL の検証**: 入力が `youtube.com` / `youtu.be` の形式かどうかをフロントまたは API でチェックし、それ以外は「YouTube の URL を入力してください」などとエラーにする。
- **エラーハンドリング**: oEmbed が 404 やタイムアウトした場合は「動画が見つからないか、URL が正しくありません」などと表示する。
- **CORS**: oEmbed はサーバー側（API Route）から叩くため、CORS の問題を避けられる。

### 3.3 フォーム項目との対応（現状の「最新動画」編集モーダル）


| フォーム項目       | 入力方法                                     |
| ------------ | ---------------------------------------- |
| **URL**      | ユーザーが貼り付け。ここをトリガーに「情報を取得」を実行できるようにする。    |
| **タイトル**     | oEmbed の `title` で自動セット。手で編集も可能。         |
| **サムネイルURL** | oEmbed の `thumbnail_url` で自動セット。手で編集も可能。 |


動画 ID は、保存時に URL から抽出して持っておくか、埋め込み用に URL から都度生成する、などの運用でよい。

## 3.4 B案で「再生」はどうするか（確認）

**B案（API Route + oEmbed）** はあくまで **「編集モーダルで URL を貼ったときに、タイトル・サムネイルを自動取得する」** ためのものであり、**再生を同一ページでするか・別タブで開くか** は別の仕様です。


| 再生の仕様                           | 必要な実装                                                   | VideoLinkPlayer.tsx の関係                                                                                                                                     |
| ------------------------------- | ------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **リンクをクリック → 別タブで YouTube を開く** | サムネイル＋タイトルを表示し、`<a href={動画URL} target="_blank">` で開く。  | 再生用の iframe や「一覧から選択して再生」の UI は不要。**再生部分は使わない**。表示は「カード＋リンク」だけでよい。                                                                                          |
| **ホームページの「最新動画」セクション内で再生する**    | 一覧（サムネイル・タイトル）を表示し、選択した動画を **同一ページ内で iframe 埋め込み再生**する。 | **そのまま有効に活用できる**。VideoLinkPlayer は「動画一覧 → 選択 → embed URL で iframe 再生」という流れになっており、表示データ（`sampleVideos`）を「管理者が登録した最新動画リスト」に差し替えれば、ホームの最新動画セクション用のプレーヤーとして使える。 |


**結論**

- **「選択した動画をホームページ内で表示（再生）する」** 場合は、**同一セクション内にプレーヤーを表示する**必要があり、そのときに **VideoLinkPlayer.tsx** の「embed URL の組み立て」「iframe 再生」「サムネイル一覧＋選択」がそのまま有効に活用できる。
- B案（oEmbed）は **編集側の「URL 貼り付け → タイトル・サムネイル取得」** の話で、**再生を別タブで開く前提にはしていない**。再生は「同一ページ内で embed」か「別タブでリンク」かを仕様で決め、同一ページで再生するなら VideoLinkPlayer を最新動画セクション用に組み込む形にする。

## 4. 次のステップ（実装する場合）

1. **API Route の追加**: 例として `app/api/youtube-oembed/route.ts` を用意し、クエリまたは body で受け取った URL を検証したうえで `https://www.youtube.com/oembed?url=...` に GET し、`title` / `thumbnail_url` 等を返す。
2. **最新動画編集モーダルの変更**: 「URL」欄の近くに「URL から情報を取得」ボタンを追加し、クリック時に上記 API を呼んで、返却値で「タイトル」「サムネイルURL」を更新する。
3. （任意）URL 欄に変更があったときだけ「情報を取得」を有効にする、または blur 時に自動取得する、などの UX 調整。

この方針で進めれば、YouTube の URL を貼り付けるだけで、付帯情報（少なくともタイトルとサムネイル）を入手する手段を用意できます。

## 5. 既存実装の再利用案

- **動画 ID 抽出・embed URL・サムネイル URL 組み立て**は、`VideoLinkPlayer.tsx` に似たロジックがすでにあるため、**共通ユーティリティ**に切り出すとよい。
  - 例: `src/lib/youtube.ts` に `getYouTubeVideoId(url)`, `getYouTubeEmbedUrl(url)`, `getYouTubeThumbnailUrl(videoId, quality?)` を定義。
  - `getYouTubeVideoId` では、既存の `embed/` と `v=` に加え、`**youtu.be/VIDEO_ID`** にも対応させる。
- **VideoLinkPlayer** と **最新動画編集モーダル（LatestVideosEditModal）** の両方でこのユーティリティを利用する。
- **タイトル・サムネイルの自動取得**だけ、新規で API Route + oEmbed を追加する（上記「案 B」）。

## 関連: 記事・いちおしサイトの OGP 取得

- **注目記事**: 記事 URL を貼り付けると **Open Graph**（og:title, og:description, og:image, og:site_name）を取得して見出し・リード・出所・サムネイルを自動入力。API: `GET /api/article-ogp?url=...`。編集モーダル: `LatestArticlesEditModal`。
- **いちおしサイト**: 同 API（`/api/article-ogp`）を流用。URL からサイト名・サムネイル・タイトルを取得。編集モーダル: `ReferenceLinksEditModal`。保存先: `site_content/home.referenceLinks`。
