# Markdown書式ガイド

## 📋 Markdownとは

Markdown（マークダウン）は、軽量マークアップ言語です。プレーンテキストで書式を記述し、HTMLやその他の形式に変換できます。

---

## 1. 基本的な書式

### 1.1 見出し（Heading）

```markdown
# 見出し1（最大）
## 見出し2
### 見出し3
#### 見出し4
##### 見出し5
###### 見出し6（最小）
```

**表示例**:
- `#` = 見出し1（最大、通常はページタイトル）
- `##` = 見出し2（セクション）
- `###` = 見出し3（サブセクション）
- `####` = 見出し4（さらに細かい分類）

**ルール**:
- `#`の数が多いほど、小さな見出しになります
- `#`の後には**スペースが必要**です（`#見出し`ではなく`# 見出し`）

### 1.2 強調（Bold, Italic）

```markdown
**太字（Bold）**
*斜体（Italic）*
***太字かつ斜体***
~~取り消し線~~
```

**表示例**:
- `**テキスト**` = **太字**
- `*テキスト*` = *斜体*
- `***テキスト***` = ***太字かつ斜体***
- `~~テキスト~~` = ~~取り消し線~~

**注意**:
- `**` は太字（Bold）
- `*` は斜体（Italic）
- アスタリスクの数で効果が変わります

### 1.3 表（Table）

```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| データ1 | データ2 | データ3 |
| データ4 | データ5 | データ6 |
```

**表示例**:
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| データ1 | データ2 | データ3 |
| データ4 | データ5 | データ6 |

**ルール**:
- `|` は列の区切り文字です
- 2行目は区切り線（`|-----|-----|-----|`）
- 区切り線の`-`は3つ以上必要です
- 列の配置を指定できます：
  - `|:---|` = 左揃え
  - `|:---:|` = 中央揃え
  - `|---:|` = 右揃え

## 2. Markdownを表示できるエディター・ツール

### 2.1 コードエディター（推奨）

#### Visual Studio Code（VS Code）
- **プラグイン**: Markdown Preview Enhanced
- **機能**: 
  - リアルタイムプレビュー
  - PDF/HTMLエクスポート
  - 数式、図表のサポート
- **表示方法**: `Ctrl+Shift+V`（プレビュー表示）

#### Cursor（現在使用中）
- **機能**: VS Codeベースなので、同じ機能が使えます
- **表示方法**: `Ctrl+Shift+V`（プレビュー表示）

#### Atom
- **プラグイン**: markdown-preview-plus
- **機能**: リアルタイムプレビュー

#### Sublime Text
- **プラグイン**: Markdown Preview
- **機能**: プレビュー表示

### 2.2 専用Markdownエディター

#### Typora（有料・推奨）
- **特徴**: WYSIWYG（What You See Is What You Get）
- **機能**: 
  - リアルタイムプレビュー
  - PDF/HTMLエクスポート
  - 数式、図表のサポート
- **価格**: 有料（約$15）

#### Mark Text（無料）
- **特徴**: オープンソース
- **機能**: リアルタイムプレビュー、PDF/HTMLエクスポート

#### Obsidian（無料・推奨）
- **特徴**: ノート作成に特化
- **機能**: 
  - リアルタイムプレビュー
  - リンク機能
  - グラフ表示

### 2.3 オンラインツール

#### GitHub
- **機能**: リポジトリ内の`.md`ファイルを自動的に表示
- **表示方法**: リポジトリで`.md`ファイルを開く

#### GitLab
- **機能**: GitHubと同様

#### Notion
- **機能**: Markdown形式で入力可能
- **特徴**: データベース機能もあり

#### Markdown Editor（オンライン）
- **URL**: https://dillinger.io/
- **機能**: ブラウザでMarkdownを編集・プレビュー

### 2.4 ブラウザ拡張機能

#### Markdown Viewer（Chrome拡張機能）
- **機能**: ブラウザで`.md`ファイルを直接表示

## 3. 各エディターでの表示方法

### 3.1 Visual Studio Code / Cursor

**プレビュー表示**:
1. `.md`ファイルを開く
2. `Ctrl+Shift+V`（Windows）または `Cmd+Shift+V`（Mac）
3. または、右上のプレビューアイコンをクリック

**サイドバイサイド表示**:
1. `Ctrl+K V`（Windows）または `Cmd+K V`（Mac）
2. 左側に編集、右側にプレビュー

**拡張機能のインストール**:
1. 拡張機能タブを開く（`Ctrl+Shift+X`）
2. "Markdown Preview Enhanced"を検索
3. インストール

### 3.2 GitHub

**表示方法**:
1. GitHubリポジトリに`.md`ファイルをアップロード
2. ファイルをクリック
3. 自動的にMarkdownとして表示されます

**編集方法**:
1. ファイルを開く
2. 鉛筆アイコン（編集）をクリック
3. Markdown形式で編集
4. プレビュータブで確認

### 3.3 オンラインツール

**Dillinger**:
1. https://dillinger.io/ にアクセス
2. 左側にMarkdownを入力
3. 右側にリアルタイムでプレビュー表示

## 4. Markdownの基本構文一覧

### 4.1 テキスト装飾

| 構文 | 表示 | 説明 |
|------|------|------|
| `**テキスト**` | **太字** | Bold |
| `*テキスト*` | *斜体* | Italic |
| `***テキスト***` | ***太字かつ斜体*** | Bold + Italic |
| `~~テキスト~~` | ~~取り消し線~~ | Strikethrough |
| `` `コード` `` | `コード` | インラインコード |
| `[リンク](URL)` | [リンク](URL) | リンク |

### 4.2 リスト

**順序なしリスト**:
```markdown
- 項目1
- 項目2
  - サブ項目1
  - サブ項目2
```

**順序付きリスト**:
```markdown
1. 項目1
2. 項目2
   1. サブ項目1
   2. サブ項目2
```

**チェックボックス**:
```markdown
- [ ] 未完了
- [x] 完了
```

### 4.3 コードブロック

**インラインコード**:
```markdown
`コード`
```

**コードブロック**:
````markdown
```言語名
コード
```
````


**例**:
````markdown
```typescript
const hello = "world";
```
````

### 4.4 引用

```markdown
> 引用文
> 複数行も可能
```

### 4.5 水平線

```markdown
---
または
***
または
___
```

### 4.6 リンク

```markdown
[リンクテキスト](URL)
[リンクテキスト](URL "ツールチップ")
```

### 4.7 画像

```markdown
![代替テキスト](画像URL)
![代替テキスト](画像URL "タイトル")
```

## 5. 表の詳細

### 5.1 基本的な表

```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| データ1 | データ2 | データ3 |
```

### 5.2 列の配置

```markdown
| 左揃え | 中央揃え | 右揃え |
|:-------|:--------:|-------:|
| データ1 | データ2 | データ3 |
```

**説明**:
- `:---` = 左揃え
- `:---:` = 中央揃え
- `---:` = 右揃え

### 5.3 表の注意点

- `|` は列の区切り文字です
- 区切り線（2行目）は必須です
- セル内に`|`を含める場合は、`\|`とエスケープします

## 6. よく使うMarkdown構文

### 6.1 見出しの階層

```markdown
# レベル1（最大）
## レベル2
### レベル3
#### レベル4
```

**推奨**:
- レベル1は通常、ページタイトルに1回だけ使用
- レベル2以降を繰り返し使用

### 6.2 強調の使い分け

```markdown
**重要**: 太字で強調
*補足説明*: 斜体で補足
`コード`: コードや技術用語
```

### 6.3 表の活用

```markdown
| 機能ID | 機能名 | 説明 | 優先度 |
|--------|--------|------|--------|
| F-001 | 機能1 | 説明1 | 高 |
| F-002 | 機能2 | 説明2 | 中 |
```

## 9. よくある質問

### Q1: `###` と `**` の違いは？

**A**: 
- `###` = 見出し（セクションのタイトル）
- `**` = 太字（テキストの強調）

**例**:
```markdown
### セクションタイトル
これは**重要な**ポイントです。
```

### Q2: `|` は何に使う？

**A**: 表（テーブル）の列の区切り文字です。

**例**:
```markdown
| 列1 | 列2 |
|-----|-----|
| データ1 | データ2 |
```

### Q3: どのエディターが一番良い？

**A**: 用途によります：
- **開発者**: VS Code / Cursor（既に使用中）
- **ドキュメント作成**: Typora または Obsidian
- **オンライン確認**: Dillinger

### Q4: Markdownはどこで使える？

**A**: 
- GitHub（自動表示）
- GitLab（自動表示）
- Notion
- Slack（一部）
- Discord（一部）
- 多くのブログプラットフォーム

## 附録: アファメーション用スロット記法（アプリ固有）

本プロジェクトでは、プロファイルの **正本はブロック式**（`sections` / `blocks`）です。**発行時**にそれを機械的に Markdown に落とし、その中でユーザー入力欄を次の形式で表します（詳細は `docs/manabiba_01/04_AFFIRMATION_DESIGN.md` §1.4〜1.6）。

| 記法 | 意味 |
|------|------|
| `[[slotId:最大文字数]]` | `slotId` に対応する入力スロット。例: `[[future.self:50]]` は最大 50 文字。 |

- 生成された Markdown 内の **通常の Markdown**（見出し、リスト、改行、強調など）は、ブロック→Markdown の変換ルールに従います。
- **発行パイプライン**では、まず blocks から上記記法を含むテキストを組み立て、続けてユーザー入力で `[[slotId:n]]` を置換し、**1 本の完成 Markdown 本文** を得ます（完成文の保存形式）。
