## 1. システム概要

### 1.1 システム名
**人生学び場 こころ道場**

### 1.2 システムの目的
AI×セルフマネジメント×伴走コーチングで、「なりたい自分」へ確実に歩むための場を提供する。

### 1.3 システムの概要
ユーザーが自己理解を深め、目標を設定し、PDCAサイクルを通じて継続的に成長をサポートするWebアプリケーション。

### 1.4 主要な価値提案
- **自己理解**: 
     - 自分らしさ(価値観、強み、願望)の明確化
     - 認知バイアス(こころのブレーキ)の明確化
- **目標設定**:
     - Be,Do,Haveフレームワークに基づいた具体的な目標設定

- **計画作成**:
     - 目標達成方法策定：情報収集、構造化、プロセス化の支援
     - リソース確保：リソースの認識・確保手段の検討支援
     - スケジューリング：マイルストーン、ガントチャート表示

- **PDCAマネジメント**: 計画→実行→確認→改善サイクルの継続的支援

- **伴走コーチング**: AIとコーチによる継続的なサポート

- **コーチングプログラムの提供**: 
    - 自分を変える7日間プログラム
      - 自己理解、目標設定までの一連の流れを７つのステップで体験
    - 28日間　行動とこころのトライアル
      - 設定した目標に対してPDCAを回し「変化」を体験する
    - 参照資料: `docs/manabiba_01/paper_based_tools/01_ワーキングシート_v0.62.txt`（詳細要件は 2.8.3 を参照）

### 1.5 対象ユーザー
- **プライマリユーザー**: 自己成長を目指す個人
- **セカンダリユーザー**: コーチ、管理者

**詳細（資料反映）**:
- 参照元: `docs/manabiba_01/paper_based_tools/01_ワーキングシート_v0.62.txt`
- **設計思想**: 7日間プログラムは、既存機能（2.2、2.3、2.4）を時系列でプロセス化したもの。各ステップは既存機能と紐づき、データも共有する。
- **7日間プログラム構成と既存機能との対応関係**:
  - Step0（インテーク）: 自分の声、聴いていますか？
  - 1日目 Step1: 望む人生を描いてみる（WS1: 理想の人生（8つの領域））
    → **2.2 自己理解機能** SELF-001「理想の人生」見える化機能（曼荼羅チャート）
  - 2日目 Step2: あなたはだれ？（WS2: 自分らしさを見つめる / WS3: 自分らしさを表現）
    → **2.2 自己理解機能** SELF-002 自分らしさ探索機能
  - 3日目 Step3: こころのブレーキ（WS4: こころのブレーキを探る）
    → **2.2 自己理解機能** SELF-003 心のブレーキ探索機能
  - 4日目 Step4: 踏み出す一歩は何？テーマを決める（WS5: 実行領域の見出し / WS6: 満足度の見直し）
    → **2.3 目標テーマ選択機能**（2.3.1 ユーザータイプ別テーマ選択機能 / 2.3.2 満足度判定機能）
  - 5日目 Step5: 望む姿を明確に（WS7: 目指すべき姿の明確化）
    → **2.4 目標設定機能** GOAL-001〜GOAL-004（SMART: S, M, R, T）
  - 6日目 Step6: 行動しなければ何も始まらない（WS8: 行動の明確化）
    → **2.4 目標設定機能** GOAL-005（Do: 行動目標）
  - 7日目 Step7: こころのブレーキを外す（WS9: あり方の選択 / WS10: 思い込みを弱める）
    → **2.4 目標設定機能** GOAL-006（Be: あり方の設定）
  - Step8: 明日からできることを始めよう（WS11: アファメーション作成シート）
    → **2.4 目標設定機能** GOAL-006（Be: あり方の設定、アファメーション策定）
- **データ共有**:
  - 各ステップで入力されたデータは、対応する既存機能のデータストレージに保存される
  - コーチングプログラムの進捗管理（COACHPROG-004）は、各機能のデータ完了状態を参照して進捗を追跡する
  - ユーザーは、コーチングプログラム経由でも、各機能から直接でも、同じデータにアクセス・編集可能
- **28日間トライアルの記録観点（例）**:
  - 朝: アファメーション宣言（した/しなかった）、今日の行動イメージング（した/しなかった）
  - 夜: 今日の行動（実行できた/一部できた/できなかった）、行動の結果、成果や変化、行動時の感情・心の動き、心のブレーキ（あった/なかった/気付かなかった）、反論の言葉、今日の気づき、今日の自分へのひとこと
  - 評価軸は「できた/できなかった」ではなく、気づきと次の改善に置く
  - 28日間トライアルの記録は、2.5 PDCAサイクル機能とデータ構造を共有する可能性がある（要検討）

- **28日間トライアル画面のUI仕様（ナビゲーション・レイアウト）**（プロトタイプで確定、実装時に準拠）:
  - **ナビゲーション**: トライアル画面（アファメーション、朝・晩、週、月）のサイドバー「ホーム」は、ホーム画面のURLへ遷移する。**実装時**は、ホーム画面側で認証状態（AuthContext／Firebase Authentication 等）を参照し、ログイン済みかどうかを内部的に判定して表示を切り替える。プロトタイプでは認証がないため、URLクエリ `logged_in=true` でログイン状態を模擬しているだけであり、本番実装ではこのパラメータに依存しない。
  - **レイアウト**: トライアル画面のメインコンテンツ領域（trial-main）の最大幅は、ホームページの左カラム（content-left）と同一とする。値は **1026px**（ホームの main-content 最大幅 1300px − 右カラム 250px − ギャップ 24px）。実装時はこの幅を共通化し、ホームとトライアルで一貫した読了幅を確保する。

**アファメーション（COACHPROG-005）詳細**（設計の正本: [04_AFFIRMATION_DESIGN.md](./04_AFFIRMATION_DESIGN.md)、データ: [03_FIRESTORE_DATABASE_STRUCTURE.md](./03_FIRESTORE_DATABASE_STRUCTURE.md)）:

- **プロファイル**: 穴埋め定義は **Firestore** に保存（システム: `affirmation_profiles/{profileId}`、ユーザー定義・改変: `users/{uid}/affirmation_profiles/{profileId}`）。現段階は固定プロファイル 1 本をコード定数でも可。**正本はブロック式**（`sections` / `blocks` のシリアル）。他プログラムでセクション・スロット単位の入力を作り最後にまとめる拡張に整合（[04_AFFIRMATION_DESIGN.md](./04_AFFIRMATION_DESIGN.md) §1.6）。
- **発行本文の形式**: 発行時に blocks から **機械生成**した **Markdown** を保存する。スロット部分の記法は **`[[slotId:最大文字数]]`**（例: `[[future.self:50]]`）。穴あき Markdown 文字列だけを DB のマスタにしない。他プログラムとの入力紐づけは `profileId` + `slotId`。
- **文書**: 発行後はプロファイルと切り離した **1 本文**（Markdown）。保存先は **`users/{uid}/affirmations/{affirmationId}`**（履歴は `history` サブコレクション）。
- **ID**: `affirmationId` は **同一ユーザ内でユニーク**。表示名の重複は UI で禁止。
- **UI**: サブメニュー **選択 / 作成 / 編集 / 履歴**。初期はプレビューのみ表示。作成はモーダル。[保存]＝下書き、[発行]＝発行済登録、[キャンセル]＝**完全破棄**。削除は **物理削除**（履歴含む・確認ダイアログ必須）。選択一覧の項目: **名称・状態・最終更新日・プロファイル名**。
- **履歴**: **発行済みのみ**対象。「履歴を残さない」保存は **上書き**。履歴閲覧は現段階 **参照のみ**。
- **暗号化**: 穴埋め入力・本文・履歴・一覧用 `lastPreviewText` は Firestore 保存前に **暗号化**（要件 7.3・プライバシーポリシーと整合）。
- **権限**: クライアント（所有者）は全操作。コーチは **発行済みかつ共有許可** の本文を閲覧（下書き・本文編集・削除・編集履歴 `history` は不可）。**A-11** でコーチコメントのデータ構造・割当を [03_A11_COACH_SHARING_SCHEMA_DRAFT.md](./03_A11_COACH_SHARING_SCHEMA_DRAFT.md) に定義（実装・ルールは同書・[04_TRIAL_28_IMPLEMENTATION_DECISIONS.md](./04_TRIAL_28_IMPLEMENTATION_DECISIONS.md) §5.1）。管理者は **IF からユーザーのアファメーション本文を閲覧しない**。
